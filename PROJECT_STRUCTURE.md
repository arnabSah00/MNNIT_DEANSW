## Key Directories Explained

### src/
Main source code directory containing all React components and logic.

### src/components/
Reusable components used across multiple pages:
- Navbar: Navigation component with mobile support
- Footer: Footer with links and contact info
- Hero: Hero banner section

### src/layouts/
Layout components that wrap pages:
- MainLayout: Contains Navbar, Outlet, and Footer

### src/routes/
Routing configuration:
- AppRoutes: All route definitions

### src/pages/
Page-level components, one folder per page:
- Home: Landing page
- Events: Event listings
- Gallery: Image gallery
- Notice: Announcements
- Contact: Contact form
- Archive: Searchable archive
- ScholarshipFellowship: Scholarship info
- MedalsAwards: Awards information
- InsuranceMediclaim: Insurance details
- AntiRagging: Anti-ragging policies
- CounsellingAssistance: Mental health support

### src/assets/
Static assets folder:
- Images
- Icons
- Logos
- Other media files

## Development Workflow

1. **Feature Development**: Create new components in appropriate folders
2. **Styling**: Use Tailwind CSS classes
3. **Routing**: Add routes in AppRoutes.jsx
4. **Navigation**: Update Navbar.jsx for new links
5. **Testing**: Use npm run dev for development testing
6. **Build**: Use npm run build for production


## Best Practices

✅ Keep components in dedicated folders
✅ Use consistent naming conventions
✅ Place styles in component files (Tailwind)
✅ Keep pages lean and import components
✅ Use React Router for navigation
✅ Follow component composition patterns
✅ Maintain consistent folder structure
